BassSfx Class

BassSfx is an extention to the BASS audio library, providing a set of functions for rendering Sonique Visualization plugins or Winamp visualization plugins on a provided device context (hDC).

Namespace: ManagedBass.Sfx
Syntax
public static class BassSfx

Properties

LastError

Declaration
public static BassSfxError LastError { get; }
Property Value
BassSfxError

If no error occured during the last BASS_SFX function call then BASS_SFX_OK is returned, else one of the BassSfxError values is returned. See the function description for an explanation of what the error code means.

Load(String)

Load this library into Memory.

Declaration
public static bool Load(string Folder = null)
Parameters
Folder
String

Directory to Load from... null (default) = Load from Current Directory.
Returns
Boolean

true, if the library loaded successfully, else false.
Remarks

An external library is loaded into memory when any of its methods are called for the first time. This results in the first method call being slower than all subsequent calls.

Some BASS libraries and add-ons may introduce new options to the main BASS lib like new parameters. But, before using these new options the respective library must be already loaded. This method can be used to make sure, that this library has been loaded.

PluginCreate(String, IntPtr, Int32, Int32, BassSfxFlags)

Creates a plugin object for use in the SFX.

Declaration
public static int PluginCreate(string File, IntPtr hWnd, int Width, int Height, BassSfxFlags Flags)
Parameters
File
String

The filename and path of the plugin to load.
hWnd
IntPtr

The handle of the window where the plugin is to be rendered.
Width
Int32

The initial width for the plugins rendering window.
Height
Int32

The initial height for the plugins rendering window.
Flags
BassSfxFlags

Any combination of BassSfxFlags.
Returns
Int32

If successful, the new SFX plugin handle is returned, else 0 is returned.
Remarks

Once you create a plugin using this function you can perform a number of different function calls on it. The Width and Height parameters are only useful for BassBox, WMP, and Sonique plugins. Winamp ignores these parameters so you can just pass in 0 for winamp plugins. Windows Media Player plugins can be created using the full path to a dll file or by using the CLSID GUID from the registry.

Mainly you might use the PluginStart(Int32), PluginSetStream(Int32, Int32) or PluginRender(Int32, Int32, IntPtr) methods.

PluginModuleGetActive(Int32)

Gets the active module for a visual plugin.

Declaration
public static int PluginModuleGetActive(int Handle)
Parameters
Handle
Int32

The SFX plugin handle (as obtained by PluginCreate(String, IntPtr, Int32, Int32, BassSfxFlags)).
Returns
Int32

If successful, the active module (zero based index) is returned, else -1 is returned.
Remarks

Visual plugins might provide multiple independent modules. You might get the number of available modules with PluginModuleGetCount(Int32). However, you can only start/activate one module at a time for a certain visual plugin.

Note: Sonique plugins only ever have 1 module. Winamp plugins can have multiple modules. So this call is really only useful for Winamp.
